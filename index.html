Redis学习总结（一）

Redis数据结构类型使用

String（字符串）

  1. set key value 设置键值对
  2. get key获取键对应的值
  3. exists key判断是否存在key
  4. append key value往key对应的值后面追加value的值，key不存在时，则新建当前key
  5. strlen key返回字符串的长度
  6. incr key使值+1
  7. decr使值-1
  8. incrby key n使值+n，设置步长为n
  9. decrby key n使值-n，设置步长为n
  10. getrange key start end截取字符串，当start为0，end为-1时，返回整个字符串
  11. setrange key offset value从偏移量的索引开始替换字符串
  12. setex key second valuesetex = set with expire，设置一个多少秒后过期的键值对
  13. setnx key value当key不存在的时候，才设置字符串
  14. mset key value [key value]批量设置字符串
  15. msetnx key value [key value]批量设置字符串，此命令是一个原子性操作，当一个存在时，剩余的key不会被设置成功
  16. mget key [key]批量获取值

List（列表）

  1. lpush key value [value]往list中push值，可一次push多个，每次往头部push
  2. lrange key 对list进行排序
  3. rpush key value [value]往list的尾部push值
  4. lpop keypop出左边的元素
  5. rpop keypop出右边的元素
  6. lindex key index取list的的第几个元素，index为索引
  7. llen key取list中的长度
  8. lrem key count value移除list中count个value值
  9. ltrim key start end将list冲start索引到end索引截取，会改变list的原本的长度
  10. lset key index value改变原来list的第index个元素，替换成为value，索引不存在则会报错
  11. linsert key before|after value newValue在原来的元素的前面后者后面插入一个新的元素，原本的list长度改变

Set（集合）

  1. sadd key value 往set中添加元素
  2. smembers key 查看set中的所有元素
  3. sismember key value 查看value值是否是set中的元素
  4. scard key 获取set集合中的元素个数
  5. srem key value [value] 移除set中的元素，可批量移除
  6. srandmember key [count] 随机获取指定个数的set中的元素，若加上count，则表示获取count个
  7. spop key 随机移除一个元素，并返回其值
  8. smove source destination member 将source的set中的member元素移动到destination的set中，source减少，destination增加
  9. sdiff key1 key2 [keyn] 查看set1中和set2中的元素的差集
  10. sinter key1 key2 [keyn] 查看set1中和set2中的元素的交集
  11. sunion key1 key2 [keyn] 查看set1中和set2中的元素的并集

Hash（哈希）

数据结构为key-

  1. hset key field value 往hash中存入一个field-value的键值对
  2. hget key field 往hash中取出field的值
  3. hset key field value [field value] 批量设置hash
  4. hget key field [field] 批量获取hash中的field的值
  5. hgetall key 一次性获取hash中的所有值
  6. hdel key field [field] 批量删除hash中的键值对
  7. hlen key 获取hash中键值对的数量
  8. hexists key field 判断hash中的键是否存在
  9. hkeys key 获取hash中的所有的键集合
  10. hvals key 获取hash中的所有的值集合
  11. hincrby key field increment 增加hash中指定键的值
  12. hdecrby key field decrement 减少hash中指定键的值
  13. hsetnx key field value 存入hash中指定的键值对，若键值对不存在，则添加；若存在，则不添加

Zset（有序集合）

  1. zadd key score value [score value]给集合添加值，score为权
  2. zrange key start end 将集合排序
  3. zrangebyscore key min max 将集合按照score排序
  4. zrem key value 移除集合中的元素
  5. zcard key 查看集合中的元素个数
  6. zrevrange key min max 倒序排序
  7. zcount key min max 获取指定区间的成员数量

#Geospatial（地理空间）

使用场景：附近的人、打车距离测算
底层实现是zset，可用zset的api操作

  1. geoadd key longitude latitude member [longitude latitude member]添加一个地理位置，对应的值分别为经度、纬度、名称。地球两极无法直接添加
  2. geopos key member [member] 获取指定名称的经度和纬度
  3. geogist key member1 member2 [unit] 返回指定的两个地点名称之间的距离，后面可选单位例如unit为km则表示单位为千米
  4. georadius key longitude latitude radius m|km|ft|mi [withdist] [withcoord] [count] 查找key中指定经纬度的，指定半径的，单位为m/km/ft/mi的存在于该key中的地理位置。withdist可选是否展示两个位置的直线距离，withcoord可选是否展示该点的经纬度，count可选展示位置的数量
  5. georadiusbymember key member radius m|km|ft|mi [withcoord]效果同上，但是区别是指定key中的位置名称
  6. geohash key member [member] 将两个位置的经纬度转换成11为的字符串，两个字符串越接近，表示两个地点的距离越近

Hyperloglog（基数）

基数：不重复的元素
使用场景：统计网页的UV（一个人访问网站多次，还是算一次）
优点：占用内存是固定的 2^64个不同的元素，只需要占用12K的空间

  1. pfadd key member [member] 存放在key中member的集合
  2. pfcount key 统计key中的集合个数
  3. pfmerge destkey sourcekey [sourcekey] 将若干个sourcekey的集合合并到destkey的目标集合中，自动去重

Bitmaps（位图）

使用场景：统计用户活跃与不活跃
数据结构：都使用二进制位来进行记录，只有0/1两种状态

  1. setbit key offset value 设置一个key，offset的地方的状态为value
  2. getbit key offset 获取当前key中指定offset的状态
  3. bitcount key 返回key中1的状态的个数

SpringBoot快速入门（一）

构建方式

spring boot项目可以通过以下几种方式搭建

  • 在spring boot网站通过maven构建下载一个压缩包，导入至IDE中
  • 使用idea快速搭建出一个spring boot项目
  • ……

这里展示两种搭建spring boot项目的方式

1.maven构建

首先访问https://start.spring.io/，project选择Maven Project，language选择Java，spring boot版本选择默认版本，项目元数据根据自己的需要填写，比如packaging方式是Jar还是War，JDK版本是8或以上等，构建方式是Maven，所以可以在dependencies中选择要依赖的东西，详细细节可以参考下图

信息填写完成后，点击绿色的Generate，或者快捷键Ctrl+回车即可下载压缩包，解压之后用IDE将文件夹通过引入Maven项目的方式引入即可。

2. idea构建

使用idea构建spring boot项目同样很方便快速，在idea中new project，左边选择Spring Initializr，右边选择JDK，下面让你选择Service URL，默认的正好就是第一种构建方式的访问地址，可以想象得到，idea其实也是通过第一种方式创建spring boot项目的。

点击next就到下一步，输入项目的元数据

再点击next，就到了选择依赖的地方，还能选择spring boot的版本号，之后再finish

这样进入idea的界面后，Maven会自动引入spring boot所需要的依赖包，一个spring boot项目就这样搭好了。

项目结构介绍

如一般的Maven/SSM工程一样，项目目录中分别含有java、resources、test的文件夹，分别用以存放Java文件、配置文件和测试文件，还有着pom.xml，但却缺少了web.xml文件和applicationContext.properties等文件，这是为什么呢？

spring boot遵循着约定优于配置的理念，特点是简单、快速和便捷。spring boot提供了一系列的Starter依赖来简化jar包的配置。

所以我们创建了一个spring boot项目之后，一切工作都如我们创建了普通的SSM框架的默认配置一样，接下来，我在包结构中创建一个controller，里面写一个简单的返回字符串的方法。

动态规划初入门

发表于2020-06-16更新于2020-07-01

动态规划

动态规划(Dynamic Programming， DP)是运筹学的一个分支，是求解决策过程最优化的过程。

动态规划作为算法题目中比较经典，也是难题之一，曾困扰我非常久。每次刷题的时候，看到动态规划求解的题型，我都唯恐避之不及。但是一直逃避也不是办法，于是我打算重✊出击，迎难而上，克服恐惧！

什么是动态规划

经过我查找多篇博客，总结出动态规划的核心思想是：记录以前做过的操作，等到下一次要再使用时，可以不用重复操作。

我们可以把一类活动分成多个互相联系的各个阶段，在每一个阶段经过操作之后得到的结果记录下来，并使用到下一个阶段之中。举个例子，就是当要去计算1+1+1+1的结果时，可以先记录第一个1和第二个1相加的结果为2，然后再将2加上第三个1结果为3，然后再将3加上最后一个1结果为4，这样一来，就符合了动态规划的思想。很明显，动态规划是一个能够通过空间换取时间的算法，将结果储存起来，避免多次计算。

动态规划问题如何解决

想要解决动态规划问题，理解了上面的概念之后，就很好办了。

其实就是要把问题划分成一个个的小问题，然后找到每一个小问题的突破口，并将这些串联起来，最后将大问题解决！

其实网上动态规划的思路非常多，但终究还是需要分成了以上几点，我这里也是借鉴了博主吴师兄的文章

看完题目之后，按照之前说的解题思路来，首先将问题分成若干个子问题，爬 n 阶楼梯，每次可以爬 1 或 2 个台阶，所以可以看成是从 n-1 阶爬上来的或者是从 n-2 阶爬上来的，而 n-1 阶可以看成是从 n-2 阶 或者是从 n-3 阶，n-2 阶可以看成是从 n-3 阶或者是 n-4 阶，以此类推，到最后就是算从第 0 阶楼梯开始爬。

然后就是找小问题的解决的出口，不管是从 n-1 阶开始还是从 n-2 阶开始，这两种情况都是不同的方法，是题目需要我们去计算的。所以到达 n 阶的方法个数是从起点到 n-1 的方法个数加上起点到 n-2 阶的方法个数，所以根据上面的结论，每一阶的方法个数都等于前两阶的方法个数之和（排除掉阶数为1的），所以能得出一个状态公式：dp[n] = dp[n-1]+dp[n-2]，当 n=0 时，不需要爬楼梯，dp[0]=0。n=1 时，只需要且只有一种方法爬到第 1 阶，所以dp[1]=1。有了这两个基石，就能推算出每一阶台阶的次数了。

这题的解题思路也能够非常清晰，我们可以使用一个数组来存储每次偷窃第 n 间房的最高金额。

遍历到第 n 间房的时候，只要判断偷与不偷的情况，比如偷第 1 间房的时候肯定是最高的，所以dp[0] = nums[0] (nums为每间房的金钱数)。

到了第 2 间房的时候，就判断是偷第 1 间划算还是偷第 2 间房划算，所以dp[1] = max(dp[0],nums[1])。

当到了第 3 间房的时候，就需要去判断是否偷第 3 间房，如果偷了第 2 间房，那就不能偷第 3 间，那么就要判断是只偷第 2 间房划算还是偷 1、3 间房划算，所以dp[2] = max(dp[0]+nums[2],dp[1])。

以此类推，就能得出一个状态方程：dp[i] = max(nums[i] + dp[i - 2], dp[i - 1]);
